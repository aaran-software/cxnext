# Contributing

## Required Workflow

1. Read `ASSIST/Documentation/` and `ASSIST/Discipline/`
2. Confirm assumptions and affected modules
3. Implement the smallest safe increment
4. Validate changed areas
5. Update documentation and changelog in the same change

## Repository Rules

Always follow:

- `ASSIST/Discipline/CODING_STANDARDS.md`
- `ASSIST/Discipline/BRANCHING_AND_COMMITS.md`
- `ASSIST/Discipline/REVIEW_CHECKLIST.md`
- `ASSIST/Discipline/TESTING_POLICY.md`
- `ASSIST/Discipline/RELEASE_DISCIPLINE.md`

## Pull Request Expectations

1. State scope clearly
2. Describe validation performed
3. Identify risks or gaps
4. Mention documentation updates
